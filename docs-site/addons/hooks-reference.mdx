This page lists all available hooks in Anki. Hooks are auto-generated from [`pylib/tools/genhooks.py`](https://github.com/ankitects/anki/blob/main/pylib/tools/genhooks.py) (backend hooks, accessed via `anki.hooks`) and [`qt/tools/genhooks_gui.py`](https://github.com/ankitects/anki/blob/main/qt/tools/genhooks_gui.py) (GUI hooks, accessed via `aqt.gui_hooks`).

See [Hooks & Filters](/addons/hooks-and-filters) for usage instructions.

### Reviewing

#### `overview_did_refresh`

**Args:** `overview: aqt.overview.Overview`

Allow to update the overview window. E.g. add the deck name in the title.

---

#### `overview_will_render_content`

**Args:** `overview: aqt.overview.Overview`, `content: aqt.overview.OverviewContent`

Used to modify HTML content sections in the overview body. `content` contains the sections of HTML content the overview body will be updated with.

When modifying the content of a particular section, please make sure your changes only perform the minimum required edits. You should avoid overwriting or interfering with existing data as much as possible, instead opting to append your own changes, e.g.:

```python
def on_overview_will_render_content(overview, content):
    content.table += "\n<div>my html</div>"
```

---

#### `overview_will_render_bottom`

**Args:** `link_handler: Callable[[str], bool]`, `links: list[list[str]]`\
**Returns:** `Callable[[str], bool]`

Allows adding buttons to the Overview bottom bar.

Append a list of strings to `links` to add new buttons:

- First value: shortcut shown in tooltip
- Second value: URL to trigger
- Third value: button text

Extend `link_handler` to handle new URLs:

```python
links.append(['H', 'hello', 'Click me!'])
def custom_link_handler(url):
    if url == 'hello':
        print('Hello World!')
    return link_handler(url=url)
return custom_link_handler
```

#### `browser_will_build_tree`

**Args:** `handled: bool`, `tree: aqt.browser.SidebarItem`, `stage: aqt.browser.SidebarStage`, `browser: aqt.browser.Browser`\
**Returns:** `bool`

Used to add or replace items in the browser sidebar tree. `tree` is the root `SidebarItem`. `stage` is an enum from `aqt.browser.SidebarStage` describing the construction stage.

Return `handled` unchanged to let Anki continue building that stage. Return `True` to prevent Anki from adding its own items at that stage. Returning `True` at `SidebarStage.ROOT` prevents the entire sidebar from being populated.

```python
def on_browser_will_build_tree(handled, root, stage, browser):
    if stage != SidebarStage.TAGS:
        return handled
    # your tag tree construction code
    # root.addChild(...)
    return True
```

### Webview

#### `webview_did_receive_js_message`

**Args:** `handled: tuple[bool, Any]`, `message: str`, `context: Any`\
**Returns:** `tuple[bool, Any]`

Used to handle `pycmd()` messages sent from JavaScript. `message` is the string passed to `pycmd()`. For messages you don't want to handle, return `handled` unchanged. Return `(True, None)` to stop the message from propagating. Return `(True, some_value)` to pass a value to `pycmd`'s result callback.

`context` is the instance passed to `set_bridge_command()` — inspect it to check which screen is firing:

```python
if not isinstance(context, aqt.reviewer.Reviewer):
    return handled
if message == "my-mark-action":
    context.onMark()
    return (True, None)
return handled
```

#### `webview_will_set_content`

**Args:** `web_content: aqt.webview.WebContent`, `context: object | None`

Used to modify web content before it is rendered. `web_content` contains the HTML, JS, and CSS the web view will be populated with. `context` is the instance passed to `stdHtml()`.

```python
def on_webview_will_set_content(web_content: WebContent, context):
    if not isinstance(context, aqt.reviewer.Reviewer):
        return
    addon_package = mw.addonManager.addonFromModule(__name__)
    web_content.css.append(f"/_addons/{addon_package}/web/my-addon.css")
    web_content.js.append(f"/_addons/{addon_package}/web/my-addon.js")
    web_content.head += "<script>console.log('my-addon')</script>"
    web_content.body += "<div id='my-addon'></div>"
```

#### `exporter_will_export`

**Args:** `export_options: aqt.import_export.exporting.ExportOptions`, `exporter: aqt.import_export.exporting.Exporter`\

def on_exporter_will_export(export_options, exporter):
    if not isinstance(exporter, ApkgExporter):
